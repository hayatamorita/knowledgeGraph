# 作業要領書
# PDF / HTML から内容を読み取り、オントロジーグラフ化し、Web UI上でグラフ表示するアプリ開発

================================================================================
1. 目的
================================================================================

本作業要領書は、ユーザーがPDFまたはHTMLを指定・アップロードし、
その内容から概念・エンティティ・関係を抽出し、
ナレッジグラフとして構造化し、
Web UI上でノード・エッジ形式のグラフとして表示するアプリを作成するための
具体的な作業手順を定義する。

本アプリの目的は以下である。

1. PDFまたはHTMLを入力として受け取る
2. 文書本文を抽出する
3. 文書内容を意味単位に分割する
4. 文書から重要な概念・用語・対象・属性を抽出する
5. 抽出した概念間の関係を抽出する
6. ノード・エッジ形式のグラフデータに変換する
7. グラフデータをDBに保存する
8. Web UIでグラフを可視化する
9. ノード・エッジをクリックして詳細・根拠テキストを確認できるようにする
10. 必要に応じてユーザーがグラフを編集・修正できるようにする

================================================================================
2. 作成するアプリの完成イメージ
================================================================================

ユーザー操作：

1. ユーザーがWeb画面を開く

CREATE TABLE documents (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  source_type TEXT NOT NULL,
  source_url TEXT,
  file_path TEXT,
  raw_text TEXT,
  cleaned_text TEXT,
  metadata JSONB DEFAULT '{}',
  processing_status TEXT NOT NULL DEFAULT 'uploaded',
  error_message TEXT,
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now()
);

CREATE TABLE document_chunks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  document_id UUID NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
  chunk_index INTEGER NOT NULL,
  heading TEXT,
  content TEXT NOT NULL,
  token_count INTEGER,
  metadata JSONB DEFAULT '{}',
  created_at TIMESTAMP DEFAULT now()
);

CREATE TABLE graph_nodes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  document_id UUID NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
  node_key TEXT NOT NULL,
  label TEXT NOT NULL,
  type TEXT NOT NULL DEFAULT 'Concept',
  description TEXT,
  properties JSONB DEFAULT '{}',
  source_chunk_ids UUID[] DEFAULT '{}',
  confidence NUMERIC DEFAULT 0.0,
  x NUMERIC,
  y NUMERIC,
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now(),
  UNIQUE(document_id, node_key)
);

CREATE TABLE graph_edges (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  document_id UUID NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
  from_node_id UUID NOT NULL REFERENCES graph_nodes(id) ON DELETE CASCADE,
  to_node_id UUID NOT NULL REFERENCES graph_nodes(id) ON DELETE CASCADE,
  relation_type TEXT NOT NULL DEFAULT 'related_to',
  label TEXT NOT NULL,
  description TEXT,
  properties JSONB DEFAULT '{}',
  source_chunk_ids UUID[] DEFAULT '{}',
  evidence_text TEXT,
  confidence NUMERIC DEFAULT 0.0,
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now()
);

CREATE TABLE extraction_jobs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  document_id UUID NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
  status TEXT NOT NULL DEFAULT 'pending',
  step TEXT,
  input_summary TEXT,
  output_summary TEXT,
  error_message TEXT,
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now()

17. Phase 6: チャンク分割機能作成
================================================================================

作業内容：

1. cleaned_textを意味単位に分割する
2. document_chunksに保存する

対象ファイル：

MVPの分割方針：

- 1チャンクあたり 800〜1500文字
- 可能なら見出し単位を優先
- overlapは100〜200文字程度
- chunk_indexを付与する

疑似コード：

function chunkText(text, maxLength = 1200, overlap = 150) {
  const chunks = [];
  let start = 0;
  let index = 0;

  while (start < text.length) {
    const end = Math.min(start + maxLength, text.length);
    const content = text.slice(start, end).trim();

    if (content.length > 0) {
      chunks.push({
        chunk_index: index,
        content
      });
      index++;
    }

    start = end - overlap;
    if (start < 0) start = 0;
    if (end === text.length) break;
  }

  return chunks;
}

完了条件：

- 文書が複数チャンクに分割される
